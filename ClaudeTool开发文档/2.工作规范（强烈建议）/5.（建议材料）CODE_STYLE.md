# CODE_STYLE — 代码规范

> 双栈项目：前端 TypeScript + 后端 Rust。两栈各有规范但共享原则。

---

## 1. 语言与本地化

- **代码标识符**：英文（变量、函数、类、文件名、目录名）
- **代码注释**：英文（便于工具处理与 PR 协作）
- **UI 文案**：中文简体，集中在 `app/src/i18n/zh-CN.ts`，**禁止**组件内硬编码中文
- **日志**：英文（`tracing` 输出便于 grep）
- **错误消息**：
  - Rust 内部 `AppError`：英文
  - 抛到 IPC 边界后由前端转中文 toast 给用户
- **文档**（`*.md`）：中文
- **Git commit**：英文，详见 §8

---

## 2. 命名约定

### 通用
- 布尔变量：`is/has/can/should` 开头（TS）；Rust 同
- 私有：TS 用 `_` 前缀（仅约定，无强制）；Rust 用 `pub(crate)` / 不加 `pub`

### TypeScript / React
| 类型 | 规则 | 示例 |
|------|------|------|
| 变量 / 函数 | `camelCase` | `loadSessions` |
| 组件 / 类 | `PascalCase` | `HistorySidebar` |
| 类型 / 接口 | `PascalCase` | `SessionMeta` |
| 常量 | `UPPER_SNAKE_CASE` | `DEFAULT_TIMEOUT_MS` |
| 文件（组件） | `PascalCase.tsx` | `Terminal.tsx` |
| 文件（工具/store） | `camelCase.ts` | `useSessionStore.ts` |
| 目录 | `kebab-case`（顶层） / `PascalCase`（组件目录） | `components/Terminal/` |

### Rust
| 类型 | 规则 | 示例 |
|------|------|------|
| 函数 / 变量 / 模块 | `snake_case` | `list_sessions` |
| 类型 / Trait / Enum | `PascalCase` | `SessionMeta` |
| 常量 / static | `UPPER_SNAKE_CASE` | `DEFAULT_TIMEOUT_MS` |
| 文件 | `snake_case.rs` | `history_parser.rs` |

## 3. 格式化

- **TS**：Prettier（行宽 100、2 空格、单引号、尾逗号 ES5、加分号），配置文件 `.prettierrc`
- **Rust**：`rustfmt`（默认配置 + `max_width = 100`），配置文件 `rustfmt.toml`
- **不允许手动调格式覆盖 formatter**
- **保存自动格式化**（VSCode 配置 `editor.formatOnSave = true`，仓库内 `.vscode/settings.json` 提交此配置）
- pre-commit hook（Husky + lint-staged）跑 `prettier --check` 与 `cargo fmt --check`

---

## 4. 注释

- **默认不写注释**——名字说清楚就不写
- **WHY 而非 WHAT**：写注释只解释"为什么这么做"，不复述"做了什么"
- **格式**：
  - TS：`// WHY: ...` 单行；多行用 JSDoc 但仅给公开 API
  - Rust：`/// ...` 给公开 API（`cargo doc` 会用），`// WHY: ...` 给非显然逻辑
- **TODO**：必须带姓名和日期：`// TODO(name, 2026-04-20): ...`
- **绝对不**留 `// removed: ...` 或 `// 这个函数做 X` 之类的注释碎片

## 5. 错误处理

### Rust
- 业务错误用 `AppError`（`thiserror::Error` 派生），定义在 `src-tauri/src/error.rs`
- 内部函数返回 `Result<T, AppError>`，用 `?` 传播
- **禁止**生产路径用 `unwrap()` / `expect()`（测试与 `main()` 启动初始化除外）
- Tauri command 边界把 `AppError` 转 `Result<T, String>`（含中文友好消息）
- 错误必须含上下文（操作名 + 关键参数），用 `thiserror` 的 `#[from]` 链上下游
- **不吞异常**，不写空 `Err(_) => ()`

### TypeScript
- 外部边界（IPC 调用、用户输入校验）必须 try/catch
- 内部函数信任类型系统，不过度防御
- 异步错误必须 await + try（不允许 unhandled promise rejection）
- 显示给用户：统一走 `useToast()` 显示中文消息
- **禁止**`catch (e) { console.log(e) }` 这类静默吞错

## 6. 类型系统

### TypeScript
- 严格模式：`strict: true`、`noUncheckedIndexedAccess: true`、`exactOptionalPropertyTypes: true`
- **禁止 `any`**，未知类型用 `unknown` + 类型守卫
- 函数必须写返回类型（除箭头函数和单行）
- 优先 `type` 别名，需扩展时用 `interface`
- 枚举优先字符串联合：`type Theme = 'light' | 'dark'`，不用 `enum`
- IPC 边界类型必须与 Rust 端 `serde` 结构对齐——共享类型放 `app/src/ipc/types.ts`，由 Rust 端用 `ts-rs` 自动生成（V2 引入）

### Rust
- 公共 API 类型必须 `#[derive(Debug, Clone, Serialize, Deserialize)]`（IPC 类型）
- 用 `Option<T>` 表达可空，不用空字符串/-1 等哨兵值
- `&str` 优先 `String`，参数侧；返回类型按所有权决定
- Lifetime 显式标注（不让编译器推断对人类不显然的）
- `clippy::pedantic` 默认开，部分 lint 按需 `#[allow]`

## 7. 测试代码风格

- **测试文件位置**：
  - Rust：与被测模块同文件 `#[cfg(test)] mod tests`，或 `tests/` 目录（集成测试）
  - TS：`*.test.ts` 与被测文件同目录
- **命名**：
  - Rust：`fn test_should_X_when_Y()`
  - TS：`describe('module')` + `it('应该 X 当 Y 时')`
- **一个测试只断言一件事**
- 共享 fixture：Rust 放 `tests/fixtures/`，TS 放 `__fixtures__/`
- **禁止**测试里 hit 真实网络或真实文件系统外部资源

---

## 8. Git 提交规范

- **格式**：`type(scope): summary`
- **type**：`feat` / `fix` / `docs` / `refactor` / `test` / `chore` / `perf` / `style`
- **scope**：模块名（`pty` / `history` / `skills` / `ui` / `env` / `net` / `config` ...）
- **summary**：英文祈使句，≤ 60 字符，结尾不加句号
- **body**（可选）：解释 WHY，不复述 WHAT；中英皆可，多人协作场景偏英文
- **示例**：
  - `feat(history): list sessions in selected workdir`
  - `fix(pty): handle ConPTY UTF-8 boundary on Windows`
  - `docs(prd): mark F3 as deferred per ADR-002`
- **禁用**：
  - `update`、`change`、`fix bug` 这类毫无信息量的 message
  - 全大写、emoji 前缀

## 9. 代码审查清单

写完代码自检，提交前必过：

### 通用
- [ ] 没有未用的 import / 变量
- [ ] 没有 `console.log` / `dbg!` / `println!`（除显式日志）
- [ ] 没有硬编码的中文（应走 i18n）
- [ ] 没有硬编码的路径（应走 `fs::claude_home()` 等）
- [ ] 没有注释掉的旧代码块
- [ ] 没有 TODO 不带姓名和日期

### TypeScript
- [ ] 没有 `any`
- [ ] 公开函数有返回类型
- [ ] 异步函数中所有 Promise 都 await 或 .catch
- [ ] 组件 props 有类型
- [ ] 列表渲染有 key

### Rust
- [ ] 生产路径无 `unwrap` / `expect`
- [ ] 错误类型实现了 `From` 链
- [ ] 公开 API 有 `///` doc
- [ ] `clippy::all` + `clippy::pedantic` 通过（或显式 `#[allow]` 并写理由）
- [ ] 跨平台代码用 `#[cfg(target_os = "...")]` 隔离

### 架构
- [ ] 没有违反 ARCHITECTURE.md §6 红线（UI 直接读文件、跨模块 import 内部等）
- [ ] 引入新依赖前已征询用户（若是）
- [ ] 关键决策有 ADR
