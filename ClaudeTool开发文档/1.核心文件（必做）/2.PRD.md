# PRD — 产品需求文档

> 立项依据见根目录 `MARKET_RESEARCH.md`。本文是开发期的功能权威源。

---

## 1. 产品定位

ClaudeTool 是一款 **Windows 优先的桌面 GUI 工具**，把 Claude Code（Anthropic 官方 CLI）包装成对小白友好的工作台。**目标用户**是听说过 Claude Code、但被 npm/WSL/PATH/网络问题劝退的中文非专业开发者，以及独立开发者、设计师、产品经理。

**当前替代方案的痛点**：
- 官方 Claude Code 是纯终端，新人不知道怎么装、不知道命令、不知道历史会话怎么续。
- Opcode（21K Star）是英文 + 仅 macOS/Linux，对中国 Windows 小白不可用。
- ccusage / Usage Monitor 都是 CLI，对小白门槛高。

**差异化**：中文 UI + Windows 优先 + 安装/网络/历史/命令/Skills 一站式 + 嵌入式终端（不替代 CLI 本体）。

## 2. 用户画像

### 画像 A（主要画像）：小李，28 岁，独立设计师 / 副业开发者
- **职业/角色**：UI 设计师，业余写小工具
- **技术水平**：会用 VSCode、不会用 WSL、看到终端命令会犹豫
- **日常工作流**：用 Cursor 写前端，想试 Claude Code 但卡在安装
- **核心痛点**：
  - 不知道 npm 全局安装失败是 PATH 问题
  - 装完输 `claude` 报"命令未找到"
  - 网络偶尔通偶尔不通，不知道是被墙还是 VPN 没开
- **使用本软件的场景**：周末试新工具，希望「下载一个 exe，点开就能用」
- **对软件的期待**：傻瓜、明确、出错时告诉他下一步该做什么

### 画像 B（次要画像）：老王，35 岁，全栈工程师

> 次要画像 ≠ 不重要。功能取舍冲突时优先小李，但日常体验细节（快捷键、终端流畅度）要兼顾老王，避免他用不下去。
- **职业/角色**：在小公司干全栈
- **技术水平**：熟练 Linux/终端
- **日常工作流**：每天用 Claude Code 写后端，开 5–10 个项目
- **核心痛点**：
  - 项目多，记不清每个项目最近聊到哪
  - 偶尔想看某个 Skill 干嘛用的，每次都要查文档
  - 终端里敲 `claude --resume` 看 picker 太慢，想直接点
- **使用本软件的场景**：日常写代码时常驻一个 ClaudeTool 窗口
- **对软件的期待**：不打扰他写代码、提供他需要时的快捷入口

## 3. 核心用户旅程

**首次启动**：
1. 双击 ClaudeTool.exe 启动
2. 应用自动检测：Claude Code 是否已安装 + 是否能访问 Anthropic
3. 任一不通过 → 弹「就绪向导」给出明确下一步
4. 全通过 → 进入主界面，提示「点左上角选个文件夹开始」

**日常使用**：
1. 左侧选择项目文件夹
2. 列表展示该项目下历史会话（按时间倒序）
3. 用户选择「新会话」或点某条历史 → 右侧终端启动 `claude` 或 `claude --resume <id>`
4. 用户在终端里和 Claude 对话
5. 想看某个命令怎么用 → 点顶栏「命令大全」，搜索后点「试一试」自动写入终端
6. 想看装了哪些 Skill → 点顶栏「Skills」

## 4. 功能清单（MVP）

### F1. Claude Code 安装检测 + 一键安装 [P0]
- **用户故事**：作为小白，我想点开软件就知道电脑里 Claude Code 是不是装好了；没装的话点一下按钮就帮我装好；装好了但还没登录的话也别让我自己摸索。
- **验收标准**：
  - 启动时检测 `claude --version`、`where claude`、`%LOCALAPPDATA%\Programs\claude\` 等位置
  - 已安装 → 显示版本号 + 绿勾，并继续调 `claude auth status`：
    - 已登录 → 直接进主界面
    - 未登录 → 弹一次性对话框「检测到未登录 Claude.ai，请在终端里执行 `claude auth login` 完成登录」+ [打开 Claude.ai] [我已登录]，**含「不再提示」选项**写到 AppConfig
  - 未安装 → 弹「安装向导」对话框：
    - 显示「未检测到 Claude Code，约需下载 80MB」+ 当前网速估算
    - 主按钮「一键安装」→ 走 Anthropic 官方 **Windows 原生安装包**路径（不走 npm，避免 Node 依赖）
    - 安装流程：下载官方 MSI/EXE → 校验签名 → 触发 UAC → 静默安装 → 自动 `claude --version` 验证
    - 全程进度条 + 当前步骤文案
    - 任一步失败 → 显示具体原因 + 「打开官方下载页手动安装」回退按钮
  - 安装成功 → 自动关闭对话框，进入主界面
  - 详细方案与回退见 ADR-009
- **依赖**：与 F4 **并行检测**，所有结果统一在 ReadinessWizard 展示；执行一键安装时需 F4 状态为可达，否则按钮置灰

### F2. 文件夹选择 + 嵌入式终端 [P0]
- **用户故事**：作为用户，我想选个项目文件夹，然后在该目录下打开一个 Claude Code 终端。
- **验收标准**：
  - 顶栏「选择文件夹」按钮 → 系统目录选择对话框
  - 选中后，右侧终端区在该路径启动 `claude`
  - 终端支持复制、粘贴、滚动、调整大小、Ctrl+C 中断
  - 切换文件夹会提示当前会话将关闭
- **依赖**：F1（必须先装 Claude Code）

### F3. ~~Token 用量监控~~ [⏸️ 暂缓]
- **状态**：暂缓，等 Anthropic 官方提供配额查询 API 再实现。详见 `12.DECISIONS.md` ADR-002。
- **理由**：第三方仅能拿到「已消耗」，无法准确反映剩余量与刷新时间。强行估算会误导小白用户，与本产品「准确、可信赖」的定位冲突。

### F4. 启动时网络可达性检测 [P0]
- **用户故事**：作为用户，我想在打开软件时就知道网络能不能用，省得对话失败才发现。
- **验收标准**：
  - 启动后 3 秒内完成检测（HEAD `https://api.anthropic.com` + DNS 解析）
  - 顶栏小圆点：绿色（可达）/ 黄色（慢，>1s）/ 红色（不通）
  - 红色时点击展开提示「请确认网络可访问 Anthropic」
  - **本工具不内置代理配置**：网络问题由用户在 OS 层面（如 VPN）自行解决，OS 级 VPN 会覆盖所有进程包含 PTY 子进程，无需本工具注入 `HTTPS_PROXY`（详见 ADR-013）
- **依赖**：无

### F5. Skills 列表 + Top 10 推荐 [P0]
- **用户故事**：作为用户，我想看看本机装了哪些 Skills，每个干嘛用，还有哪些值得装。
- **验收标准**：
  - 扫描 `~/.claude/skills/`、`~/.claude/plugins/*/skills/`、`<workdir>/.claude/skills/`
  - 列表展示：名称、来源、描述、`SKILL.md` 链接
  - 点击某个 Skill → 侧栏展开完整 `SKILL.md`
  - 「推荐安装」标签页展示 MVP 内置的 30 个种子 Skill 中**未安装**的 Top 10（按内置 Star 数排序）
  - 每条推荐附「复制安装命令」按钮（暂不做一键安装）
- **依赖**：F2（需要 workdir 路径）

### F6. 历史对话面板 + 一键续聊 [P0]
- **用户故事**：作为用户，我想看到当前项目下所有过去的对话，点一下就能继续聊。
- **验收标准**：
  - 选完文件夹后，左侧列表展示该项目下所有 `.jsonl` 会话
  - 每条显示：会话名（无名时用 summary 或首条 prompt 截断）、最近时间、轮次
  - 点击 → 在终端启动 `claude --resume <sessionId>` 并自动回车
  - 提供「继续最近一次」（`claude -c`）和「Fork 一份」（`--fork-session`）快捷按钮
  - 支持模糊搜索
- **依赖**：F2

### F7. Claude Code 命令大全 [P0]
- **用户故事**：作为新人，我不知道 Claude Code 有哪些命令、参数，希望有个地方查。
- **验收标准**：
  - 顶栏「命令大全」按钮 → 抽屉式面板
  - 内置中文化命令字典：**MVP ≥ 30 条覆盖最高频场景**（V1.1 补到 100 条），按用途分类（会话/认证/插件/MCP/调试/权限）
  - 每条含：命令、中文用途、示例、坑点
  - 搜索框模糊匹配（命令名 + 中文关键字）
  - 「试一试」按钮 → 把命令写入终端但不自动回车，让用户确认后执行
- **依赖**：F2（需要终端可写）

### F8. 对话方向引导 [⏸️ V2]
- **状态**：MVP 不做。V2 起步走规则引擎方案（见调研报告 §3 需求 8）。

### F9. 半自动更新提示 [P1]
- **用户故事**：作为用户，我希望知道有新版本不用每次都去 GitHub 看；但我也不想被强制自动更新覆盖掉当前用得好好的版本。
- **验收标准**：
  - 启动时静默调 GitHub Releases API（5s 超时；失败静默跳过，不打扰用户）
  - 发现新版 → 顶栏齿轮按钮加红点
  - SettingsDialog 内置「检查更新」按钮，点击触发一次同步检测，含 toast 反馈
  - 用户点红点 / 检查更新 → 弹小弹窗「v1.X.X 可用，[查看更新内容] [前往下载]」
  - [前往下载] → 浏览器开 GitHub Releases 页
  - **不自动下载、不自动安装**（避免无签名带来的安全风险，与 Q2.2 决策一致）
- **依赖**：F4（网络可达；不通时本次启动跳过此检查）

## 5. 不做什么（Out of Scope）

- ❌ MVP **不做** Token 配额监控（等官方 API）
- ❌ MVP **不做** npm 路径的安装（Node 缺失需自行安装的复杂分支留 V1.1，详见 ADR-009）
- ❌ MVP **不做**对话方向引导（V2）
- ❌ MVP **不做** Skills 一键安装，只复制命令
- ❌ **永远不做**：内置代理 / 替代 Claude Code 自身的对话能力 / 上传任何用户数据 / 推广绕过区域限制
- ❌ MVP 仅 Windows，macOS 和 Linux 留 V1.1
- ❌ MVP **不做**多账号 / 多 profile 切换
- ❌ MVP **不做**编辑器、文件浏览器（让用户用自己熟悉的 IDE）

---

## 6. 成功指标

- 安装到首次成功对话 ≤ 5 分钟（小白可用性）
- 启动到主界面可交互 ≤ **3 秒**
- 网络与安装检测准确率 ≥ 95%
- 历史对话续聊一键命中率 ≥ 99%（id 不会传错）
- 安装包 ≤ **250MB**（含 Bundled WebView2 离线运行时约 150–180MB；本工具自身 ≤ 30MB）
- 内存占用空闲 ≤ **150MB**
