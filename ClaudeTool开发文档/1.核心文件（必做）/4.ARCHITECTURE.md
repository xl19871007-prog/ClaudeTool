## 2. 模块划分

### 模块：`ui`（前端 React 应用）
- **职责**：渲染所有界面、捕获用户操作、订阅 Rust 端事件
- **对外接口**：通过 Tauri `invoke()` 调用 Rust commands；通过 `listen()` 订阅事件
- **依赖**：仅 Tauri IPC，**禁止**直接读文件、起进程
- **不负责**：业务逻辑（解析 jsonl、起 PTY 等）必须由 Rust 端实现

### 模块:`env_checker`（Rust）
- **职责**：检测 Claude Code 是否安装、版本号、安装位置、登录状态；执行一键安装（Win 原生安装包路径）；查询本工具自身新版本
- **对外接口**：
  - `check_claude_installed() -> Result<ClaudeStatus>`
  - `check_claude_auth_status() -> Result<AuthStatus>`（调用 `claude auth status` 解析）
  - `check_network() -> Result<NetworkStatus>`
  - `check_for_updates() -> Result<UpdateInfo>`（调 GitHub Releases API，5s 超时，失败返回 None）
  - `install_claude_native() -> EventStream<InstallProgress>`（流式进度：下载 → 校验 → 安装 → 验证）
  - `cancel_install()`
- **依赖**：`fs`、`net`
- **不负责**：
  - npm 路径的安装（V1.1）
  - Node.js 安装（V1.1）
  - PATH 手动改写（依赖官方安装包自身处理）

### 模块：`pty_manager`（Rust）
- **职责**：管理 PTY 会话的生命周期，桥接 xterm.js 与 `claude` 进程
- **对外接口**：
  - `pty_spawn(cwd, cmd, env_overrides?) -> SessionHandle`（`env_overrides` 保留作通用扩展点；MVP 不传）
  - `pty_write(session, data)` / `pty_resize(session, cols, rows)` / `pty_kill(session)`
  - 事件：`pty_output(session, data)` / `pty_exit(session, code)`
- **依赖**：`tauri-plugin-pty`
- **不负责**：
  - 终端 UI 渲染（xterm.js 在前端）
  - 代理环境变量注入（ADR-013：不内置代理，由 OS 层 VPN 解决）

### 模块：`history_parser`（Rust）
- **职责**：扫描和解析 `~/.claude/projects/<encoded-cwd>/*.jsonl`，输出会话列表与元数据
- **对外接口**：
  - `list_sessions_quick(workdir) -> Vec<SessionMeta>`（**流式读** jsonl，遇到关键字段即停；`turnCount` 字段可能为初始估值，详见 ADR-014）
  - `refine_session_metadata(session_ids) -> EventStream<SessionMetaPatch>`（后台异步精确化 `turnCount` 等慢字段；前端订阅事件局部更新列表）
  - `watch_active_session(session_id) -> EventStream`（V2 用，对话引导基础设施）
- **依赖**：`fs`、`notify`
- **不负责**：发起 resume（由 ui 层组合 `pty_spawn` 完成）

### 模块：`skills_scanner`（Rust）
- **职责**：扫描本地已装 Skills、加载内置推荐种子、合并去重
- **对外接口**：
  - `list_installed_skills(workdir) -> Vec<SkillMeta>`
  - `list_recommended_skills() -> Vec<SkillMeta>`（读内置静态 JSON）
  - `read_skill_md(path) -> String`
- **依赖**：`fs`
- **不负责**：在线安装 Skill（V2）

### 模块：`net`（Rust 基础）
- **职责**：HTTP 探测、统一 reqwest 客户端、超时与错误归一化
- **对外接口**：`probe(url) -> ProbeResult` / `client() -> reqwest::Client`
- **不负责**：业务级网络逻辑

### 模块：`config`（Rust 基础）
- **职责**：读写本工具自身配置（代理、UI 偏好、语言等），存到 `<app_data>/ClaudeTool/config.json`
- **对外接口**：`load() -> Config` / `save(config) -> Result<()>`
- **不负责**：业务数据持久化（本项目几乎无业务数据，主要消费 `~/.claude/`）

### 模块：`fs`（Rust 基础）
- **职责**：跨平台路径解析（拿 `~/.claude/`、appdata 等）、文件读写薄包装
- **对外接口**：`claude_home() -> PathBuf` / `project_session_dir(workdir) -> PathBuf`

## 4. 关键数据流

### 数据流 A：启动时检测
1. App 启动 → React mount
2. UI 调用 `invoke('check_environment')`
3. `commands::env::check_environment` **并行**调用：
   - `env_checker::check_claude_installed()` → 拿版本
   - `env_checker::check_claude_auth_status()` → 拿登录状态（仅当 installed 时有意义）
   - `env_checker::check_network()` → HEAD api.anthropic.com
   - `env_checker::check_for_updates()` → GitHub Releases API（5s 超时）
4. 结果通过 IPC 一次性返回前端
5. UI 顶栏渲染：环境/网络指示灯 + 自身新版本红点（如有）
6. 若 install/auth/network 任一异常 → 弹 ReadinessWizard，所有问题一次展示，按用户解决顺序刷新（Q3.2 决策）

### 数据流 B：选文件夹 → 起终端
1. 用户点「选择文件夹」
2. 前端调 `dialog.open({ directory: true })` 拿到路径
3. 前端调 `invoke('list_sessions', { workdir })` → 拿历史会话列表，渲染左侧栏
4. 用户点「新建会话」按钮 → `invoke('pty_spawn', { cwd: workdir, cmd: 'claude' })`
5. Rust `pty_manager` 启动子进程，分配 session_id，返回前端
6. 前端 xterm.js 实例与该 session_id 双向桥接：
   - 用户敲键 → `invoke('pty_write', { session, data })`
   - PTY 输出 → `event::pty_output` → xterm.js 渲染
   - 窗口 resize → `invoke('pty_resize', { session, cols, rows })`

### 数据流 C：点历史会话续聊
1. 用户在左侧栏点某条会话
2. 前端拿到该 `sessionId`
3. 前端调 `invoke('pty_spawn', { cwd, cmd: 'claude', args: ['--resume', sessionId], env })`
4. 之后流程同 B

### 数据流 D：命令大全「试一试」
1. 用户在命令面板搜索 → 选中某条
2. 点「试一试」
3. 前端调 `invoke('pty_write', { session: 当前活跃 session, data: '<命令文本>' })`
4. **不自动回车**，让用户校对参数后自己按回车

## 5. 扩展点

- **新平台支持**：`fs::claude_home()` / `pty_manager` 内已用 cfg(target_os) 隔离；新增平台只改这两处
- **新会话源**：`history_parser` 暴露 `SessionSource` trait，未来支持 Claude Code Web 会话只需新增实现
- **新 Skills 数据源**：`skills_scanner::list_recommended_skills()` 返回 `Vec<SkillMeta>`，V2 接 marketplace 时只换实现
- **命令大全数据**：`src/data/commands.json`（前端静态资源），V2 改为 Rust 端动态拉 `claude --help` 解析

---

## 6. 架构红线

- ❌ UI 层禁止直接调系统命令、读文件、发 HTTP（必须经 Tauri commands）
- ❌ Rust 端禁止持有 LLM API 调用逻辑（本工具**不**直连 Anthropic）
- ❌ 模块之间只通过公开接口，不 import 内部 mod
- ❌ 任何模块不得依赖 ui 层
- ❌ `pty_manager` 之外的代码不得直接 spawn 子进程
- ❌ 不得在 Rust 代码硬编码 `~/.claude/...` 路径，必须走 `fs::claude_home()`

---

## 7. 跨领域关注点

- **错误**：Rust 全部模块抛 `AppError`（`thiserror`），到 command 边界转 `Result<T, String>`（含中文提示），UI 层统一转 toast
- **日志**：Rust 用 `tracing`，输出到 `<app_data>/ClaudeTool/logs/YYYY-MM-DD.log`，按天滚动，最多保留 7 天
- **i18n**：MVP 仅中文，文案集中在 `src/i18n/zh-CN.ts`，禁止组件内硬编码中文（V2 加 i18n-next 即可切英文）
- **配置热更新**：UI 偏好（主题、字号）即时生效；网络检测开关变更下次启动生效
- **崩溃恢复**：PTY 子进程崩溃 → `event::pty_exit` 通知前端，xterm.js 显示「Claude 已退出（exit code N），点击重启」
- **兼容性提示**：检测到 `claude --version` 超出本工具已测的版本范围时，前端顶部显示横幅「检测到 Claude Code vX，本工具已测范围 Y–Z，可能不稳定」（ADR-011）
