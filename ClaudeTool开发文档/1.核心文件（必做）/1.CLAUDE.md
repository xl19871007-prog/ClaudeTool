# CLAUDE.md — 项目总纲

> 这是 ClaudeTool 项目的总纲，Claude 每次启动会优先阅读此文件。细节分流到子文档。

---

## 1. 项目一句话

**ClaudeTool** 是面向中文小白用户的 Claude Code 桌面助手，把"安装环境、嵌入式终端、历史会话、命令大全、Skills 管理、网络检测"打包成开箱即用的 GUI，让不懂 npm/WSL/PATH 的人也能 30 秒内用上 Claude Code。

---

## 2. 当前阶段

**MVP 开发中**（2026-04-20 起，目标 6–8 周）

- 优先 Windows 平台（小白用户最大盘）
- 全中文 UI
- 不做编辑器、不做模型管理、不做账号管理（这些走原生 Claude Code）

---

## 3. 核心原则（不可违反）

1. **数据准确性优先**：无可信数据源的功能宁可不做（已据此暂缓「Token 剩余量」需求 3，等官方 API）。
2. **不主动绕过区域限制**：网络检测失败只提示用户检查网络，不内置/推荐任何代理工具，不在文档/UI 中出现"翻墙""镜像""绕过"等词。
3. **本地优先**：所有用户数据（手稿、配置、Skills 列表、对话历史）只存本地，不上传任何服务器；不收集后台遥测。V2 可加「用户主动点按钮才上报」的兼容性反馈（详见 ADR-011），仍非后台采集。
4. **不重新发明 Claude Code**：本工具是"包装器与助手"，**不替代** Claude CLI 本体；任何对话内容必须通过原生 `claude` 进程，不直连 Anthropic API。
5. **小白可读**：所有 UI 文案、错误提示、引导话术使用中文，避免行话；遇到必须保留的英文术语（如 `Skill` `MCP`）须配中文说明。
6. **代码英文、文案中文**：标识符、注释、Git commit 用英文；UI 文案、用户面向的错误用中文。
7. **不引入未经 ADR 讨论的第三方依赖**：见 `12.DECISIONS.md`。
8. **永远不接入其他 LLM 提供商**：本工具定位是「Claude Code 的小白入口」（ADR-010），不做 OpenAI/Gemini/本地模型支持，不做编辑器，不做项目管理与团队协作。

---

## 4. 协作方式偏好

- 新功能开始前先出实现计划，确认后再写代码
- 一次只做一个功能，不顺手重构
- 遇到歧义先问，不要猜
- 用中文回复，代码标识符用英文
- 所有架构级决策必须先更新 `12.DECISIONS.md` 再动代码

## 6. 快速上下文

ClaudeTool 是一个 **Tauri 2 桌面应用**，前端 React+TS，后端 Rust。它本身**不与 Anthropic API 直接通信**——所有 LLM 对话能力来自用户机器上已安装的官方 `claude` CLI（Claude Code），本工具用 `tauri-plugin-pty` 把 CLI 嵌入 GUI 终端，并叠加四类增强：

1. **环境助手**：检测 Claude Code 安装、**一键安装**（Win 原生安装包路径）、检测网络、注入代理环境变量到 PTY。
2. **会话助手**：扫描 `~/.claude/projects/<encoded-cwd>/*.jsonl` 展示历史会话，点击注入 `claude --resume <id>` 续聊。
3. **知识助手**：内置中文化 Claude Code 命令大全，可点击"试一试"自动写入终端。
4. **生态助手**：扫描本地已装 Skills，展示 Top 10 推荐（MVP 用静态种子）。

V1 暂不做的：Token 配额监控（需求 3，等官方 API）、对话方向引导（需求 8，规则版排 V2）、一键安装的复杂分支（V2）。
