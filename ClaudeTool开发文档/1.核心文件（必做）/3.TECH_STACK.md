# TECH_STACK — 技术选型

> 选型理由的关键决策见 `12.DECISIONS.md`（特别是 ADR-001：Tauri vs Electron）。

## 1. 技术栈总览

| 层 | 技术 | 版本 | 为什么选 |
|---|---|---|---|
| 桌面壳 | **Tauri** | 2.x | 内存 ~30–60MB（Electron 200–300MB），启动快 40%，包体小 ~96%；与"轻量小白工具"定位匹配（ADR-001） |
| Rust 工具链 | **rustc / cargo** | stable 1.78+ | Tauri 2 要求 |
| 前端框架 | **React** | 18.x | 生态成熟、Tauri 模板默认、小白工具组件需求量大 |
| 类型 | **TypeScript** | 5.x | 强类型 + 与 Tauri 命令接口契合 |
| 构建 | **Vite** | 5.x | Tauri 模板默认 |
| UI 组件 | **shadcn/ui + Tailwind CSS** | 最新 | 复制式组件、可定制、上手快 |
| 状态管理 | **Zustand** | 4.x | 轻量、无 boilerplate、Tauri 项目主流 |
| 路由 | **React Router** | 6.x | 主界面只有 1–2 个路由，轻量足矣 |
| 终端渲染 | **xterm.js** | 5.x | 事实标准；Opcode、VSCode、Hyper 都用它 |
| xterm 插件 | xterm-addon-fit / xterm-addon-web-links | 最新 | 自适应尺寸、URL 可点击 |
| PTY | **tauri-plugin-pty**（Rust crate） | 最新 | Tauri 原生，Win 用 ConPTY，避免 Node sidecar |
| 文件监听 | **notify**（Rust crate） | 6.x | jsonl 增量监听 |
| HTTP 探测 | **reqwest**（Rust crate） | 0.12.x | 网络可达性检测，避免反病毒拦截 |
| 序列化 | **serde / serde_json** | 1.x | Rust 端解析 jsonl |
| 日志 | **tracing**（Rust） / **pino**（前端可选） | 最新 | Rust 标准日志栈 |
| 单元测试 | **Vitest**（前端） / **cargo test**（Rust） | 最新 | 框架原生 |
| E2E 测试 | **Playwright** | 最新 | 跨平台，可驱动 Tauri WebView |
| Lint / 格式 | **ESLint + Prettier**（TS） / **clippy + rustfmt**（Rust） | 最新 | 标准组合 |
| 包管理 | **pnpm**（前端） / **cargo**（Rust） | 最新 | pnpm 节省磁盘、装得快 |

## 2. 运行环境

- **操作系统**：
  - **MVP**：Windows 10 1809+（x64）
  - **V1.1**：macOS 12+（Intel + Apple Silicon），Linux（Ubuntu 22.04+ 作为参考发行版）
- **运行时**：无需用户预装（Tauri 打包时含所有依赖）
- **WebView**：
  - Windows：WebView2，**采用 Bundled 模式打包**（详见 ADR-012）—— 安装包内嵌 WebView2 离线运行时（约 150–180MB），所有 Win10/11 版本无网络也可装
  - macOS：WKWebView（系统自带）
  - Linux：WebKitGTK（需用户系统装 `webkit2gtk`）
- **最低硬件**：4GB RAM、500MB 可用磁盘、屏幕 1280×720+
- **网络要求**：
  - 软件本身：可离线启动
  - 软件功能：需用户自行确保能访问 `api.anthropic.com` 和 `claude.ai`（Claude Code 官方要求）

## 3. 依赖管理原则

- 优先用 Rust / TS 标准库与 Tauri 内置能力，能不引入依赖就不引入
- 新增依赖前必须评估：包大小、维护活跃度（最近 6 个月有 commit）、license（仅 MIT/Apache-2.0/BSD）、周下载量
- **首次引入新依赖必须先征询用户**，并在 `12.DECISIONS.md` 留 ADR
- 安全相关依赖必须锁精确版本号
- 禁用停止维护超过 12 个月的包
- Rust 端用 `cargo audit`，前端用 `pnpm audit`，高危漏洞零容忍

---

## 4. 禁用技术

- ❌ **Electron**：内存与包体劣势明显（ADR-001）
- ❌ **Node sidecar 跑 PTY**：用 Rust 端 `tauri-plugin-pty`，避免双运行时
- ❌ **`any`（TypeScript）**：用 `unknown` + 类型守卫
- ❌ **`unwrap()` 在生产路径上的 Rust 代码**：必须 `?` 或显式 match，仅测试和初始化允许
- ❌ **moment.js**：用 `dayjs`
- ❌ **直接 fetch Anthropic API**：本工具**不与 Anthropic API 通信**，所有 LLM 请求经由用户机器上的 `claude` CLI（CLAUDE.md 核心原则 #4）
- ❌ **GPL/AGPL 依赖**：与 MIT 主项目 license 冲突
- ❌ **遥测 / 埋点 SDK**：MVP 不收集任何用户数据
- ❌ **任何代理 / VPN 库**：CLAUDE.md 核心原则 #2

## 5. 关键库的使用约定

- **HTTP 请求**：Rust 端统一走 `src-tauri/src/net/client.rs` 包装的 reqwest 实例（含超时、UA、错误归一化）
- **日志**：Rust 用 `tracing`，前端用 `console`（dev）/ Tauri 桥接到文件（prod），禁止散落 `println!` / `dbg!`
- **路径**：Rust 端用 `dirs` crate 拿用户目录（跨平台），不硬编码
- **JSON**：Rust 端 `serde_json`，前端原生 `JSON.parse`，不引入 lodash 等
- **状态**：前端跨组件状态走 Zustand store；组件本地状态用 React useState；不使用 Context 跨多层
- **Tauri command**：Rust 暴露给前端的命令统一在 `src-tauri/src/commands/` 下按域分模块（`commands/skills.rs`、`commands/sessions.rs` 等）
- **错误**：Rust 用 `thiserror` 定义 `AppError`，前端 catch 后统一转中文 toast

## 6. 版本升级策略

- **补丁版本**（`x.y.Z`）：自动升级，CI 通过即合
- **次版本**（`x.Y.z`）：手动升级，PR 中说明影响面
- **主版本**（`X.y.z`）：必须开 ADR 讨论，特别是 Tauri / React 大版本
- 锁文件（`Cargo.lock` / `pnpm-lock.yaml`）必须提交
