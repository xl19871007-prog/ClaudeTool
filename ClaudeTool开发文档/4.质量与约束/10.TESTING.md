# TESTING — 测试策略

> 双栈项目：前端 Vitest + Testing Library，后端 cargo test，端到端 Playwright。

---

## 1. 测试金字塔

- **单元测试 70%**：Rust 模块纯函数（`history_parser` / `skills_scanner` / `env_checker` / `config`）+ 前端 store 与工具函数
- **集成测试 20%**：Rust 端跨模块（如 `pty_manager` + `config`）+ 前端组件与 store 交互
- **端到端 10%**：Playwright 跑 Tauri 应用，验证 PRD 中的关键用户旅程

倒金字塔（多 E2E 少单元）会被 review 拒绝——E2E 慢、脆、调试难。

---

## 2. 测试工具

| 类型 | 工具 | 配置文件 |
|------|------|---------|
| Rust 单元/集成 | `cargo test`（内置） | `Cargo.toml` |
| Rust 覆盖率 | `cargo-tarpaulin` 或 `cargo-llvm-cov` | CI 配置 |
| Rust mock | `mockall` 或手写 trait + 测试实现 | — |
| TS 单元 | Vitest | `vitest.config.ts` |
| TS 组件 | `@testing-library/react` | 同上 |
| TS 覆盖率 | Vitest 内置（V8） | 同上 |
| TS mock | Vitest 内置 `vi.mock` | — |
| E2E | Playwright（Tauri WebDriver） | `playwright.config.ts` |
| Lint | ESLint + clippy | `.eslintrc.json` / `clippy.toml` |
| Pre-commit | Husky + lint-staged | `.husky/pre-commit` |

## 3. 必测清单

以下代码**必须**有测试，写代码同时写测试：

### Rust
- [ ] `history_parser`：jsonl 解析、损坏文件容错、cwd 编解码
- [ ] `skills_scanner`：三类来源合并、去重、优先级
- [ ] `env_checker`：跨平台 PATH 探测、版本号解析、网络 probe 超时
- [ ] `config`：读写、迁移、原子替换、损坏恢复
- [ ] `fs`：跨平台 `claude_home()` 路径解析（Win/Mac/Linux 各一）
- [ ] 任何 `AppError` 转换链
- [ ] Tauri command 边界的输入校验

### TypeScript
- [ ] Zustand stores 的 reducer 逻辑
- [ ] `ipc/` 层的错误转换（Rust `String` → 中文 toast）
- [ ] 命令大全的搜索/过滤函数
- [ ] 历史会话列表的排序/搜索
- [ ] 任何涉及时间格式化的工具函数

### 必测的回归场景
- 出过的 bug 必须配回归测试，避免重犯

---

## 4. 免测清单

避免过度测试：

- 纯展示组件（无逻辑、无状态）
- shadcn/ui 直接复用、未改动的组件
- 简单 getter/setter
- 一次性脚本
- `main.rs` / `App.tsx` 的胶水代码

---

## 5. 测试命名

- **Rust**：
  - `mod tests` 内函数：`fn test_<scenario>()`
  - 例：`fn test_parses_session_with_missing_summary()`
  - 例：`fn test_returns_empty_when_workdir_has_no_sessions()`
- **TS**：
  - `describe('ModuleName')` + `it('应该 + 行为 + 当 + 条件')`
  - 例：`it('应该按 updatedAt 倒序排序当展示历史时')`
  - 例：`it('应该返回空列表当 workdir 不存在时')`

---

## 6. 测试数据

- **Rust fixtures**：`src-tauri/tests/fixtures/`，含真实 jsonl 样本（脱敏）
- **TS fixtures**：`app/src/__fixtures__/`
- **factory 函数**：测试中创建数据用 factory 而非大对象字面量
  - 例：`makeSession({ updatedAt: '2026-04-19', turnCount: 5 })`
- **Mock 外部依赖**：HTTP（reqwest 用 `wiremock` 或测试 trait）、文件系统（用 `tempfile` crate）、子进程（PTY 测试用 echo / cat 等无害进程）
- **禁止**测试中 hit 真实网络或真实 `~/.claude/`

## 7. 覆盖率目标

| 模块 | 目标 |
|------|------|
| `history_parser` | ≥ 90% |
| `skills_scanner` | ≥ 90% |
| `env_checker` | ≥ 85% |
| `config` | ≥ 90% |
| `fs` | ≥ 80% |
| `pty_manager` | ≥ 70%（PTY 难单测，靠集成测） |
| 前端 stores | ≥ 80% |
| 前端组件 | 不强求覆盖率，重要交互必须有测 |
| **整体** | ≥ **70%** |

CI 中覆盖率下降阻塞合并（基于 main 分支基线）。

## 8. 手动测试清单

每次 release 前手动过：

### 安装与启动
- [ ] 全新 Win11 机器装包能启动
- [ ] 没装 Claude Code 时弹引导
- [ ] 没装 WebView2 的旧 Win10 能 bootstrap 安装

### 网络
- [ ] 网络通时顶栏绿灯
- [ ] 拔网线后顶栏红灯
- [ ] OS 层 VPN 开启时检测与 PTY 内对话均正常
- [ ] 区域被限的 403 报错正确归类（不误报为"无网络"）

### 终端
- [ ] 中文输入法（搜狗 / 微信 / 系统）正常
- [ ] 大段输出不卡（10MB 文本粘贴）
- [ ] resize 窗口 PTY 同步
- [ ] Ctrl+C 中断 Claude 进程
- [ ] 子进程 exit 后 UI 提示重启

### 历史与续聊

- [ ] 损坏的 jsonl 不崩溃

### Skills
- [ ] 用户级 / 项目级 / 插件级 都能扫到
- [ ] 重复 Skill 按优先级合并
- [ ] 推荐 Top 10 排除已装

### 命令大全
- [ ] 搜索中英文都能命中
- [ ] 「试一试」写入终端但不自动回车
- [ ] 多 Tab 切换状态保留

### 视觉与可访问性
- [ ] 深色模式无闪烁
- [ ] 100% / 150% / 200% DPI 缩放正常
- [ ] Tab 焦点顺序合理
- [ ] 屏幕阅读器（NVDA）能读到主要按钮

## 10. CI 流水线

GitHub Actions 在 PR 上跑：

1. **Lint**（并行）：`cargo clippy -- -D warnings` + `pnpm lint` + `cargo fmt --check` + `prettier --check`
2. **单元 / 集成**：`cargo test` + `pnpm test`
3. **覆盖率**：上传到 Codecov，下降阻塞
4. **Build**（仅 main 分支与 release tag）：`pnpm tauri build` 跨平台 matrix（先 Win，V1.1 加 mac/linux）
5. **E2E**（仅 nightly）：Playwright 全量场景

合并要求：
- 所有 1–3 步绿
- Reviewer 至少 1 人
- 涉及 ARCHITECTURE 改动须有对应 ADR
