# CONSTRAINTS — 非功能约束

> 这些指标是硬要求，超阈值要么优化要么开 ADR 调整。

---

## 1. 性能

- **冷启动**：双击图标到主界面可交互 ≤ **3 秒**（Win10 SSD）
- **环境检测**：启动后 3 秒内完成（含网络探测，超时 2 秒；F1 与 F4 并行）
- **历史会话扫描（流式）**：单项目 100 个会话拿到首屏元数据 ≤ **300ms**；`turnCount` 等慢字段后台异步精确化（ADR-014），不阻塞首屏
- **Skills 扫描**：100 个 Skill ≤ **200ms**
- **PTY 输出延迟**：从 `claude` 进程 stdout 到 xterm 渲染 ≤ **30ms**
- **键盘输入延迟**：用户按键到 PTY 写入 ≤ **16ms**
- **窗口 resize PTY 同步**：≤ **50ms**
- **命令大全搜索**：500 条字典内模糊匹配 ≤ **100ms**

---

## 2. 资源占用

- **空闲内存**：≤ **150MB**（不含 PTY 子进程的 `claude`）
- **打开文件数**：常态 ≤ 50（log 滚动 + 配置 + jsonl 短读）
- **磁盘占用**：安装包 ≤ **250MB**（含 Bundled WebView2 离线运行时 ~150–180MB）；本工具自身二进制部分 ≤ 30MB；运行时数据（log + config）≤ 50MB
- **空闲 CPU**：≤ **1%**
- **PTY 高吞吐场景**（Claude 大量输出）CPU ≤ 30%

---

## 3. 安全

- **不直连 Anthropic API**：所有 LLM 请求经由用户机器上的 `claude` CLI（CLAUDE.md #4）
- **不存储任何账号凭据**：用户 OAuth / API key 由 Claude Code 自己管理
- **不管理代理 / VPN**：本工具不持有任何代理相关配置（ADR-013）；如未来引入需保密字段必须用系统 keychain（Win Credential Manager / macOS Keychain）
- **PTY 输入输出不持久化**：本工具不记录用户与 Claude 的对话内容（jsonl 由 Claude Code 自己写）
- **外部输入校验**：
  - 文件夹路径必须存在且为目录
  - 会话 ID 必须匹配 UUID 正则
  - 命令大全的"试一试"只写入终端文本，**不**自动回车
- **依赖**：Rust `cargo audit` + 前端 `pnpm audit`，CI 高危漏洞阻塞合并
- **代码签名**：MVP 发布暂不做 EV 证书签名（成本高），用 SmartScreen 友好的方式：稳定的发行者名 + 声誉积累。V1.1 评估签名
- **一键安装下载校验**：F1 下载 Anthropic 官方安装包必须校验数字签名（验证发行者为 "Anthropic, PBC" 等官方主体），校验失败立即中止并提示用户走手动下载回退；不允许"忽略警告"选项
- **UAC 处理**：一键安装触发 UAC 弹窗时，本工具进程不请求自身提权，只 spawn 安装包让 Win 自己弹 UAC；用户拒绝 UAC 走回退路径，不重试

---

## 4. 隐私

- **零数据收集**：MVP **不**接入任何遥测、埋点、错误上报第三方 SDK
- **崩溃日志**：仅本地 `<app_data>/ClaudeTool/logs/`，不自动上报；用户可手动「导出诊断包」（V2）
- **日志内容**：禁止写入 PTY 输入输出原文、用户手稿、文件路径中的用户名（Win 下做脱敏）
- **网络请求**：仅以下出站请求被允许：
  - `HEAD https://api.anthropic.com`（启动检测，F4）
  - `HEAD https://claude.ai`（启动检测，F4）
  - `GET https://api.github.com/repos/<owner>/ClaudeTool/releases/latest`（启动时静默查更新，F9；5s 超时，失败静默）
  - F1 一键安装时下载 Anthropic 官方 Windows 安装包（具体 URL 由 Anthropic 官方维护，本工具读取已知列表）
  - 用户主动点的"打开下载页 / 打开 Claude.ai / 打开 GitHub Release"等浏览器跳转（不算应用内请求）
- **用户可一键清除**：`<app_data>/ClaudeTool/` 删掉即彻底卸载（不留注册表残留）

---

## 5. 可用性

- **键盘可用**：所有交互可键盘完成（Tab 顺序合理、有焦点环）
- **对比度** ≥ **WCAG AA**
- **屏幕阅读器**：图标按钮必须有 `aria-label`（中文）
- **MVP 仅中文**，但代码预留 i18n 接口（文案走 `src/i18n/zh-CN.ts`）
- **支持主流输入法**：终端区必须正确处理搜狗、微信输入法、系统输入法的 IME composition 事件（xterm.js 默认支持，需测）

---

## 6. 兼容性

- **Windows 10 1809+**（x64） — MVP 必须
- **Windows 11**（x64 / arm64） — MVP 必须
- **macOS 12+**（Intel + Apple Silicon） — V1.1
- **Linux**（Ubuntu 22.04+ 参考；其他主流发行版尽力） — V1.1
- **WebView2**：Win 10 21H2+ 自带；旧 Win10 由 Tauri bootstrapper 提示安装
- **屏幕**：最小 1280×720，自适应到 4K
- **DPI**：支持 100%–300% 缩放
- **Claude Code 兼容版本**：MVP 起最低支持 Claude Code 2.0+；维护"已测版本范围"白名单（在代码中常量定义），版本超出范围时 UI 显示兼容性横幅（ADR-011）；每次 Claude Code 大版本发布前完成兼容性回归并更新白名单

---

## 7. 可维护性

- **单文件 ≤ 400 行**（前端组件 + Rust 模块）
- **单函数 ≤ 60 行**（Rust 含 doc comment）
- **圈复杂度 ≤ 10**
- **依赖深度 ≤ 3**（不要 A → B → C → D → E）
- **公开 API 必须有类型标注**（TS）/ 完整签名（Rust）
- **架构红线见 ARCHITECTURE.md §6**

---

## 8. 可观测性

- **日志级别**：error / warn / info / debug
- **常态日志**：仅 info 及以上写文件
- **Debug 模式**：通过环境变量 `CLAUDETOOL_LOG=debug` 开启
- **关键操作必打 info**：
  - 启动 / 退出
  - 环境检测结果
  - 网络检测结果
  - PTY spawn / kill / exit code
  - 配置变更
- **异常必打 error**（含栈与上下文）
- **日志滚动**：按天滚动，保留 7 天
- **崩溃**：Tauri 内置 panic handler 写到日志；前端 React ErrorBoundary 捕获后展示「出错了，已记录到日志」
