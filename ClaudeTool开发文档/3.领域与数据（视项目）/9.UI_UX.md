# UI_UX — 界面与交互

> 用 shadcn/ui + Tailwind 实现，主界面单页，弹窗/抽屉为辅。

---

## 1. 设计原则

1. **小白一眼能懂**：核心操作最多 2 次点击；新人首次启动不超过 3 个引导步骤就能进入主界面
2. **不打断用户对话**：终端是核心区，所有辅助面板（命令大全、Skills）都用抽屉/侧栏不挡终端
3. **状态可见**：环境/网络/活跃会话状态常驻顶栏小图标，hover 出详情
4. **错误必有下一步**：任何错误提示必须含「我该怎么办」按钮，不能只丢报错
5. **键盘可达**：所有交互可键盘完成（Tab 顺序合理、有焦点环、提供常用快捷键）
6. **拒绝花哨**：动画 ≤ 200ms，无 confetti / 弹跳球这类装饰

**深色模式**：默认跟随系统（`prefers-color-scheme`），可在设置里手动覆盖。

### 字体

- **正文**：系统默认（`-apple-system, "Segoe UI", "Microsoft YaHei", "PingFang SC", sans-serif`）
- **终端 / 等宽**：`'Cascadia Code', 'JetBrains Mono', Consolas, 'Courier New', monospace`，连字开
- **字号梯度**：12 / 13 / 14 / 16 / 18 / 20 / 24（rem 体系，根 16px）

### 间距（4px 基准，Tailwind 默认）
- xs: 4 / sm: 8 / md: 12 / lg: 16 / xl: 24 / 2xl: 32

### 圆角
- sm: 4px / md: 6px / lg: 8px / 全圆: 9999px（按钮/胶囊）

### 阴影（轻、不抢视觉）
- card：`0 1px 2px rgba(0,0,0,0.05)`
- popover：`0 4px 12px rgba(0,0,0,0.10)`

## 6. 交互规则

- **保存**：所有设置即时保存（Zustand persist），不需要"保存"按钮
- **删除**：本工具 MVP **不提供**删除任何用户数据的入口（jsonl 由 Claude Code 管，本工具不碰）
- **加载**：超过 200ms 显示 skeleton；超过 2s 显示进度条；不允许卡死
- **Toast**：成功 / 信息 / 警告 显示 3 秒，错误 显示 5 秒并附「查看日志」按钮
- **空状态**：必须有引导文案 + 主行动按钮（例：未选文件夹时展示大按钮「选择项目文件夹」）
- **兼容性横幅**：检测到 Claude Code 版本超出本工具已测范围时，主界面顶部显示一条黄色横幅「⚠️ 检测到 Claude Code vX，本工具已测范围 vY–vZ，部分功能可能异常」+ 关闭按钮（关闭后本次启动不再显示）
- **快捷键**（MVP）：
  - `Ctrl+O` 打开文件夹
  - `Ctrl+K` 打开命令大全
  - `Ctrl+,` 打开设置
  - `Ctrl+L` 清屏（透传给终端）
  - `Ctrl+C` / `Ctrl+V` 在终端区按 xterm.js 默认行为（选中复制 / 粘贴）
- **Tab 顺序**：顶栏左→右；侧栏上→下；终端最后
- **焦点环**：所有可聚焦元素必须可见焦点环（`outline: 2px solid var(--accent)`）

## 7. 可访问性

- 所有图标按钮必须有 `aria-label`（中文）
- 状态小圆点用 `role="status"` + `aria-live="polite"`
- 对比度全页 ≥ WCAG AA（深色模式同等）
- 终端区禁止屏蔽屏幕阅读器（xterm.js 提供 `screenReaderMode`，开启）
- 不使用纯色彩传达信息（如错误必须图标 + 颜色 + 文字）
