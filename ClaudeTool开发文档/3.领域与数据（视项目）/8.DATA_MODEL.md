# DATA_MODEL — 数据模型

> 本工具几乎不持有"业务数据"，主要是**消费方**：读 Claude Code 在 `~/.claude/` 下的数据 + 维护一份自己的轻量配置。

---

## 1. 实体关系图

```
[用户机器]
   │
   ├── ~/.claude/                          ← Claude Code 拥有，本工具只读
   │     ├── projects/
   │     │     └── <encoded-cwd>/
   │     │           └── <sessionId>.jsonl  → SessionFile
   │     ├── skills/<skill>/SKILL.md       → SkillFile (用户级)
   │     └── plugins/<plugin>/skills/...   → SkillFile (插件)
   │
   └── <app_data>/ClaudeTool/              ← 本工具拥有
         ├── config.json                    → AppConfig
         ├── seed-skills.json               → SkillSeed[] (打包内置)
         └── logs/YYYY-MM-DD.log
```

**关系**：
- 一个 `Project`（即 cwd） 对多个 `Session`
- 一个 `Session` 是单个 jsonl 文件，含 N 条 `Turn`
- `SkillFile` 与项目无强关系，按目录归属（用户级 / 插件级 / 项目级）

---

## 2. 实体定义

### Entity: `SessionMeta`（来自 jsonl 头部 + 文件元数据）

**描述**：一条 Claude Code 历史会话的展示元数据，由 `history_parser` 解析输出给前端。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string (UUID) | ✓ | 等于 jsonl 文件名（去 `.jsonl`） |
| `name` | string \| null | ✗ | 用户用 `claude -n` 起的别名，jsonl 内可能存 |
| `summary` | string \| null | ✗ | Claude 自动生成的 summary（jsonl 第一条系统消息可能含） |
| `firstPrompt` | string | ✓ | 第一条 user 消息内容（截断 200 字） |
| `cwd` | string | ✓ | 工作目录绝对路径（解码自父目录名） |
| `turnCount` | number | ✓ | 该会话总轮次（首次返回为流式估值，后台异步精确化后通过 `refine_session_metadata` 事件刷新；详见 ADR-014） |
| `createdAt` | ISO 8601 string | ✓ | 第一条消息时间 |
| `updatedAt` | ISO 8601 string | ✓ | 最后修改 mtime |
| `filePath` | string | ✓ | jsonl 完整路径（不暴露给 UI，调试用） |
| `bytes` | number | ✓ | 文件字节数 |

**约束**：
- `id` 必须是合法 UUID
- 排序：`updatedAt DESC`
- 每条至少有 `firstPrompt`，否则视为损坏会话跳过

**生命周期**：
- 创建：Claude Code 启动新会话时由它创建
- 删除：用户手动删 jsonl，本工具**不做**删除（避免误删）

### Entity: `SkillMeta`（解析自 `SKILL.md` frontmatter）

**描述**：一个 Skill 的展示元数据。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✓ | 路径哈希（去重用） |
| `name` | string | ✓ | YAML frontmatter `name` |
| `description` | string | ✓ | YAML frontmatter `description` |
| `source` | enum `'user' \| 'project' \| 'plugin' \| 'recommend'` | ✓ | 来源分类 |
| `pluginName` | string \| null | ✗ | 来源是 plugin 时的插件名 |
| `path` | string | ✓ | `SKILL.md` 绝对路径（recommend 类型为种子 ID） |
| `installed` | boolean | ✓ | 是否本机已装 |
| `stars` | number \| null | ✗ | 推荐种子的 GitHub Star 数（仅 recommend 类型） |

**约束**：
- 同名同源去重；`source` 优先级 `project > user > plugin > recommend`（合并展示时保留最高）

### Entity: `Turn`（jsonl 单行解析后）

**描述**：会话中的一轮交互。**MVP 不展示**，仅在 V2 对话引导时使用。

| 字段 | 类型 | 说明 |
|------|------|------|
| `role` | `'user' \| 'assistant' \| 'system' \| 'tool'` | 消息角色 |
| `content` | string \| object | 内容（可能含工具调用结构） |
| `timestamp` | ISO 8601 string | 时间戳 |
| `tokensIn` | number? | 输入 token（如 jsonl 含） |
| `tokensOut` | number? | 输出 token |
| `model` | string? | 用的模型 |

> **注意**：jsonl 字段名以 Claude Code 实际版本为准，可能含 `type`、`message`、`tool_use_id`、`tool_result_id` 等，不要硬编码假设。`history_parser` 必须容错跳过未知字段。

### Entity: `AppConfig`（本工具自身配置）

**描述**：本工具的用户偏好，存 `<app_data>/ClaudeTool/config.json`。

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `version` | number | 1 | schema 版本，迁移用 |
| `lastWorkdir` | string \| null | null | 上次打开的项目目录 |
| `theme` | `'system' \| 'light' \| 'dark'` | `'system'` | UI 主题 |
| `suppressLoginPrompt` | boolean | false | F1 中"未登录"提示的"不再提示"勾选 |
| `lastSeenVersion` | string \| null | null | 用户已看过的最新版本号（避免红点重复提醒） |
| `terminal.fontSize` | number | 14 | 终端字号 |
| `terminal.fontFamily` | string | `'Cascadia Code, Consolas, monospace'` | 终端字体 |
| `commandPanel.lastQuery` | string | `''` | 命令面板上次搜索词 |
| `network.checkOnStartup` | boolean | true | 启动时检测网络 |
| `network.endpoint` | string | `'https://api.anthropic.com'` | 检测目标 |

**约束**：
- 字段缺失时用默认值，不报错（向前兼容）
- 写入用原子替换（先写 `.tmp` 再 rename）

### Entity: `SkillSeed`（打包内置的推荐种子）

**描述**：随软件分发的静态推荐 Skills 列表，文件 `app/src-tauri/src/seed/seed-skills.json`。

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | string | Skill 名 |
| `description` | string | 中文化描述（人工编写） |
| `repoUrl` | string | GitHub 仓库 URL |
| `installCmd` | string | 一行可复制的安装命令 |
| `stars` | number | 录入时的 Star 数 |
| `category` | string | 分类（`coding` / `writing` / `data` / `productivity` 等） |
| `recordedAt` | ISO 8601 string | 数据录入时间（提醒过期） |

---

## 3. 关系与级联规则

- 本工具不持有外键关系（不是数据库系统）
- 删除 `AppConfig` → 下次启动重建为默认
- 用户删除 `~/.claude/projects/<dir>/<id>.jsonl` → 本工具下次扫描时自动消失，无需级联

## 5. 存储格式

| 数据 | 存储位置 | 格式 | 谁拥有 |
|------|---------|------|--------|
| 会话历史 | `~/.claude/projects/<encoded>/<id>.jsonl` | JSON Lines | Claude Code（只读） |
| Skills | `~/.claude/skills/`、`~/.claude/plugins/` | 目录 + Markdown | Claude Code（只读） |
| 本工具配置 | `<app_data>/ClaudeTool/config.json` | JSON | 本工具 |
| 推荐种子 | `<app_data>/ClaudeTool/seed-skills.json`（首次启动从可执行内拷贝） | JSON | 本工具（只读运行时） |
| 日志 | `<app_data>/ClaudeTool/logs/YYYY-MM-DD.log` | 纯文本（tracing 默认） | 本工具 |
| 命令大全字典 | `app/src/data/commands.zh-CN.json` | JSON（打包进前端 bundle） | 本工具 |

**`<app_data>` 跨平台路径**：
- Windows: `%APPDATA%\ClaudeTool\`
- macOS: `~/Library/Application Support/ClaudeTool/`
- Linux: `~/.config/ClaudeTool/`

由 `dirs::config_dir()` Rust crate 解析。

---

## 6. 迁移策略

- `AppConfig` 含 `version` 字段；版本升级时按 `version` 走 `migrate_v1_to_v2()` 链
- 检测到不识别的高版本（用户回滚软件） → 提示「配置版本过新，将重置」并备份原文件到 `config.json.bak.<timestamp>`
- 不做数据库迁移（无数据库）
- 推荐种子（`seed-skills.json`）随版本更新覆盖，不做迁移
