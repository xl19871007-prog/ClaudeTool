# DOMAIN — 领域知识

> 本工具的"领域"是 **Claude Code 生态本身**。给 Claude（开发助手）和未来贡献者一份完整术语对照，避免代码与文案中术语错乱。

---

## 1. 领域概述

**Claude Code** 是 Anthropic 于 2025 年发布的官方 CLI 工具（仓库 `anthropics/claude-code`），让 Claude 大模型直接在用户终端里读写代码、执行命令、管理项目。它本质是一个 Node 应用（同时提供 Windows 原生安装包），通过订阅制（Pro / Max / Max 5x / Max 20x）或 API key 接入 Claude 模型。

围绕 Claude Code 形成了几个生态层：
- **Skills**：可复用的指令集，教 Claude 怎么完成特定任务（类似"个性 + SOP"）
- **Plugins**：捆绑一组 Skills + 可选的 Hooks / MCP 服务器，作为整体分发
- **MCP（Model Context Protocol）服务器**：给 Claude 接外部工具与 API
- **Hooks**：在 Claude 生命周期的关键时刻（用户提交 prompt、工具调用前后等）触发用户脚本
- **Slash 命令**：在会话内输入 `/xxx` 触发的功能（`/init`、`/resume`、`/compact` 等）

本工具 **ClaudeTool** 是上述生态的客户端包装器，**不属于** Claude Code 本身。

## 3. 业务规则

- **会话存储位置稳定**：`~/.claude/projects/<URL编码的cwd>/<sessionId>.jsonl`。Claude Code 多次升级未变，可放心解析。
- **Plugin ≠ Skill**：安装单位是 **plugin**，每个 plugin 内含一个或多个 **skill**。例如 `document-skills` plugin 含 `xlsx` / `docx` / `pptx` / `pdf` 共 4 个 skill。`claude plugin install <skill-name>` 会失败——必须用 plugin 名。
- **Marketplace ID ≠ GitHub repo 路径**：`claude plugin marketplace add <owner>/<repo>` 注册时，注册名取自该仓库 `.claude-plugin/marketplace.json` 里的 `name` 字段，而不是 GitHub repo 路径。比如官方 `anthropics/skills` 仓库注册后名字是 `anthropic-agent-skills`，安装命令必须是 `claude plugin install <plugin>@anthropic-agent-skills`。
- **Windows 上 plugin 命令需要 git-bash**：`claude plugin install` 等命令在原生 Windows 终端会报「needs git-bash」。解决方式是设环境变量 `CLAUDE_CODE_GIT_BASH_PATH` 指向 `bash.exe`（Git for Windows 自带）。M4 起本工具会自动检测并配置（ADR-017）。
- **嵌入式终端是 Claude 会话，不是裸 shell**：本工具的 xterm 跑的是 `claude`，所以：
  - **斜杠命令** `/xxx`（如 `/help` `/plugin install`）→ 是会话原生指令，**直接输入即可**
  - **shell 命令**（如 `claude --version` `git status` `ls`）→ 必须加 `!` 前缀（如 `!claude --version`），否则会被 Claude 当成对话内容
  - **plugin 安装**优先用 `/plugin marketplace add ...` + `/plugin install ...` 两步 slash，而**不是** `claude plugin install ...`（后者在会话内会被误解读）
  - 本工具的「试一试」按钮会自动判断：以 `/` 开头直接注入，其他自动加 `!` 前缀
- **会话 ID 是 UUID**：本工具传给 `--resume` 必须是合法 UUID（或用 `-n` 起的 name）。
- **Claude Code 不暴露配额查询 API**：消耗可推算，剩余量不可知（详见 PRD F3 与 ADR-002）。
- **Anthropic 区域限制**：API 直接拒绝来自不支持地区的请求，常见错误 400 `Access to Anthropic models is not allowed from unsupported countries`。本工具检测到此错误时只提示用户检查网络，**不**给出绕过建议。
- **CLI 与 Web 走不同 endpoint**：`claude` CLI 不会自动继承系统代理，必须显式传 `HTTPS_PROXY` 环境变量给子进程。
- **Skills 加载顺序**：项目级（`<workdir>/.claude/skills/`）覆盖用户级（`~/.claude/skills/`）。
- **`SKILL.md` 格式**：YAML frontmatter（`name` / `description` / `type`）+ Markdown 正文。本工具按此解析。
- **已装 plugin 的真实位置不要猜**：实际路径是 `~/.claude/plugins/cache/<marketplace>/<plugin>/<version-sha>/skills/<skill>/SKILL.md`，含动态版本目录。**权威来源**是 `~/.claude/plugins/installed_plugins.json`，里面每个 plugin 条目含 `installPath` 字段。本工具的 skills_scanner 必须读这个 manifest，**不要**遍历 `~/.claude/plugins/` 下子目录。

## 4. 典型场景

- **小白首次使用**：双击 ClaudeTool.exe → 弹「未检测到 Claude Code，点这里下载」→ 装完回来 → 「网络不通，点这里看怎么办」→ 解决后选个文件夹 → 在终端里聊起来
- **老用户日常**：常驻 ClaudeTool 窗口 → 切项目时左栏直接看到历史会话 → 想接昨天那个 → 一键续聊
- **学命令**：在会话里想 compact 上下文但忘了命令 → 顶栏点「命令大全」→ 搜「上下文」→ 找到 `/compact` → 点「试一试」→ 命令出现在终端，按回车执行
- **找新 Skill**：装完几个 Skill 想看看还有什么好用的 → 顶栏「Skills → 推荐」→ 看 Top 10 → 复制安装命令到终端

## 5. 同领域现有方案

| 工具 | 形态 | 与本工具的差异 |
|---|---|---|
| Claude Code（CLI 本体） | 终端 | 我们包装它、不替代它 |
| Claude Code Desktop（官方） | Electron | 闭源，不解决中国/小白痛点 |
| Opcode | Tauri 2 GUI（21K Star） | 仅 macOS/Linux，英文，无安装/网络/Skills 助手 |
| ccusage / Claude-Code-Usage-Monitor | CLI | 仅用量监控，命令行 |
| claude-usage Dashboard | 本地 Web 仪表盘 | 单一功能 |
| raine/claude-history | CLI | 仅历史搜索 |
| Cursor / Continue / Cline | IDE 内 AI | 编辑器集成方向，与本工具正交 |

**我们的差异**：不做编辑器、不做 LLM 直连、专注做 Claude Code 的"环境与生态"GUI 助手；中文 + Windows 优先。
